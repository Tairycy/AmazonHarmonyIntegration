bt_a2dp_abort_stream 
brief Suspend a stream. ingroup a2dp
details This function tries to suspend a stream by sending a request to the remote party. The stream can be in any state state except AVDTP_STREAM_STATE_IDLE. As a result of this operation the A2DP_EVT_ABORT_STREAM_COMPLETED event will be generated. This operation cannot be rejected. The p evt_param.abort_stream_requested.err_code is always == AVDTP_ERROR_SUCCESS.
param mgr A2DP manager. param strm_handle Stream handle.
return li c TRUE if the function succeeds, i.e. the actual request has been sent to the remote party. li c FALSE otherwise. No events will be generated.
