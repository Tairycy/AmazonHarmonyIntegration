Setup

The CTR can operate in one of the following modes:

  • WIFI mode
  • USB mode
  • GPIO mode.
the CTR hardware must be set up according to the required operation mode. 

 

Example: CTR Setup 

To initialize the CTR module for USB SoF triggers in US mode:

Copy Code
/* Counter Initialization */
/* The clock divider parameters N and M, Counter increment step LSB and the Mode for us counter is configured in Counter initialization */
/* Counter CTR0 Initialization */
PLIB_CTR_NValueSet(CTR_ID_0, CTR0, N);
PLIB_CTR_NValueSet(CTR_ID_0, CTR0, M);
PLIB_CTR_LSBValueSet(CTR_ID_0, CTR0, LSB);
PLIB_CTR_CTRModeSelect(CTR_ID_0, CTR0, Mode);

/* Latch Initialization */
/* Configures Latch trigger source, Counter and interrupts*/
/* For Wifi, Only Latch3 interrupt is enabled as Latch 3 contains the timestamp for last Wifi event (IEEE 802l.11v). For USB and GPIO, only one latch is used to store the timestamp */

 /* USB SOF */
 PLIB_CTR_LatchTriggerSelect(CTR_ID_0, CTR_LATCH0, CTR_USBSOF);
 PLIB_CTR_LatchCTRSelect(CTR_ID_0, CTR_LATCH0, CTR_CTR0_US);
 PLIB_CTR_LatchDivSet(CTR_ID_0, CTR_LATCH0, 2);
 PLIB_CTR_IntModeLatchSelect(CTR_ID_0, 0, CTR_LATCH0);
 PLIB_CTR_IntLatchSelect(CTR_ID_0, CTR_LATCH0);

/* Trigger Initialization */
/* Trigger counter and the phase are configured in trigger initialization */
PLIB_CTR_TriggerSelect(CTR_ID_0, CTR_USBSOF);
PLIB_CTR_CycleOffsetValueSet(CTR_ID_0, 0);
