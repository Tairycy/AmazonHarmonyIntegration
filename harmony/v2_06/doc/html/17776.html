Hardware Abstraction Model

To understand the hardware abstraction model for the MCPWM Peripheral Library (PLIB), it is important to understand the hardware block diagram of the MCPWM peripheral, as shown in the following figure. 

The MCPWM SFRs can be classified into two categories:

  • Module-level SFRs – These SFRs control the functionality of the peripheral which is common to all the generators in the module.
  • Generator-level SFRs – These SFRs control the functionality of individual PWM generator and hence each generator has its own instance of generator level SFRs.

MCPWM Peripheral Block Diagram
 

To correctly set up the MCPWM PLIB, the user application needs to configure the module-level SFRs and generator level SFRs corresponding to the number of PWM generators used in the application. The PLIB APIs are designed such that they address all of the SFRs associated with a logical function. As shown in the following figure, the MCPWM PLIB APIs are mainly divided into Module-level functions and Generator level functions. Within each category, all the APIs pertaining to a logical function are grouped together.
