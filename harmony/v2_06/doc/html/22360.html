Adding an Event to the Aria Quickstart Demonstration

After you have built, loaded, and run the basic application, you can explore how to add a simple event to the demonstration’s button. For details on how to load, build, program, and run this project, see Volume 1: Getting Started With MPLAB Harmony > Applications Help > Graphics Demonstrations > Demonstrations > aria_quickstart. 

The steps are as follows:

  1. In the Projects view of the project, right click aria_quickstart and set it as the main project:

 

 

  1. Launch the MPLAB Harmony Configurator:

 

 

  1. Open the project’s MHC configuration:

 

 

  1. Select the Graphics Composer from the MHC Pull-Down Menu, which will open a new window for the MPLAB Harmony Graphics Composer (MHGC):

 

 

  1. We will change the asserted behavior of the screen’s button by adding “Pressed” and “Released” events. The “Pressed” event will change the text of the button. First we must add the needed text. Select the Asset menu and click Strings.

  1. Click the Add icon, , and name the new string “Ouch_String”.
  2. Select TimesNewRoman12 as the font. Enter the string value of “Ouch! Ouch! Ouch!”. When you are done, the screen should show:

When finished, close the String Asset dialog.

 

  1. Select the Graphics Composer Screen Designer panel: and select the ButtonWidget1 (“Make changes. Generate. Run”).
  2. Under the Graphics Composer Properties panel on the right side of the screen you should see the properties of ButtonWidget1. Enable the “Pressed” and “Released” events:

  1. Click the button on the right side to bring up the Events Editor for the “Pressed” event and create a new action. The Action Edit dialog appears. Enter the name ButtonPressed and click Next.
  2. Now you need to select which widget the Pressed event of ButtonWidget1 affects, which can be any of the display's widgets. However, we want the Pressed event tochange the text of its own widget. Therefore, select ButtonWidget1 and click Next.
  3. The next dialog window instructs you to select the action associated with this event. There are many choices, from “Adjust Position” to “Set Y Position”, all of which are related to properties of the button widget. Select “Set Text” as the action, and then click Next.
  4. Select the Ouch_String you provided in step 7, and then click Finish. This action returns you to the Event Editor dialog. Click OK to close this dialog.
  5. Follow the same steps for the “Released” event, but instead create a new event named ButtonReleased. Use the same Set Text action as before, but use the Instructions string instead.
  6. Examine the events you've created by the Window:Event Manager menu. The Event Manager dialog is an alternative way of creating and managing graphics events for the application.

 

 

  1. Generate code for this new project configuration by selecting the Generate icon ( ). Save the new configuration, since it has been modified. Select “Automatically Overwrite User Changes” as the Merging Strategy, since you have not changed any MHC options. The select the “Generate” button to generate new code to implement the events you have just added.
  2. Select the Run Main Project icon ( ) to build, load, and run this new configuration.
  3. After the project has loaded, click the “Make changes. Generate. Run.” button. The button’s text should change when pressed and revert back when released.
