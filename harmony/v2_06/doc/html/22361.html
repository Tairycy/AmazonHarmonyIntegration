Creating New Graphics Applications

This tutorial provides information on how to create a new touch input-enabled graphics project using the MPLAB Harmony Graphics Composer (MHGC). At the end of this tutorial you will have created a new project equivalent to a configuration found in the aria_quickstart demonstration project. 

The following hardware configurations are supported:

  • PIC32MZ Embedded Graphics with External DRAM (DA) Starter Kit and the Multimedia Expansion Board II (MEB II)
  • PIC32MZ Embedded Graphics with Internal DRAM (DA) Starter Kit and the Multimedia Expansion Board II (MEB II)
  • PIC32MZ Embedded Graphics with Disabled DRAM (DA) Starter Kit and the Multimedia Expansion Board II (MEB II)
  • PIC32MZ Embedded Connectivity with Floating Point Unit (EF) Starter Kit and the Multimedia Expansion Board II (MEB II)
Getting Started

Follow the tutorial found in Volume 1: Getting Started With MPLAB Harmony Libraries and Applications > MPLAB Harmony Tutorial - Creating Your First Project > Part I: Creating Your First MPLAB Harmony Project in MHC to first learn about the basics of software development using MPLAB Harmony. Successfully completing the tutorial will verify your software setup is correct. 

You will need the following hardware, depending on which hardware setup you are using:

  • PIC32MZ Embedded Graphics with External DRAM (DA) Starter Kit and a Multimedia Expansion Board II (MEB II). For details on how to assemble the boards or for links on how to order the hardware, see pic32mz_da_sk_extddr+meb2
  • PIC32MZ Embedded Graphics with Internal DRAM (DA) Starter Kit and a Multimedia Expansion Board II (MEB II). For details on how to assemble the boards or for links on how to order the hardware, see pic32mz_da_sk_intddr+meb2
  • PIC32MZ Graphics (DA) Starter Kit w/DDR-disabled and a Multimedia Expansion Board II (MEB II). For details on how to assemble the boards or for links on how to order the hardware, see pic32mz_da_sk_noddr+meb2
  • PIC32MZ Embedded Connectivity (EF) Starter Kit and a Multimedia Expansion Board II (MEB II). For details on how to assemble the boards or for links on how to order the hardware, see pic32mz_ef_sk+meb2

 

Configuring the Hardware

PIC32MZ EF Starter Kit connected to the Multimedia Expansion Board II (MEB II)

  • The J9 Jumper on the MEB II board, located beneath the PIC32MZ EF Starter Kit, should be configured to support using internal SRAM for the graphics frame buffer, as shown in the following figure.
  • Connect the PIC32MZ EF Starter Kit board to the MEB II board
  • Power up the board by connecting the power adapter to J3 power connector on the MEB II board or a powered USB cable to the USB DEBUG port on the Starter Kit board

 

 

PIC32MZ Embedded Graphics with External DRAM (DA) Starter Kit and a Multimedia Expansion Board II (MEB II), 

PIC32MZ Embedded Graphics with Internal DRAM (DA) Starter Kit and a Multimedia Expansion Board II (MEB II), 

PIC32MZ Embedded Graphics with Disabled DRAM (DA) Starter Kit and a Multimedia Expansion Board II (MEB II) 

 

  • These configurations require that the J9 jumper be set to provide the GLCD's pixel clock. Set the J9 jumper to connect EPIOE to LCD_PCLK.
  • The J9 jumper is located on the bottom of the MEB II board, beneath where the starter kit is plugged into the board. Refer to the following figure for the exact location.
  • Connect the PIC32MZ DA Starter Kit board to the MEB II board
  • Power up the board by connecting the power adapter to J3 power connector on the MEB II board or a powered USB cable to the USB DEBUG port on the Starter Kit board

 

 

Tutorial Steps
  1. Start MPLAB X IDE and select File > New Project. The New Project dialog appears.
  2. In the New Project dialog, ensure that Microchip Embedded is selected, and that the project type is 32-bit MPLAB Harmony Project, as shown in the following figure, and then click Next.

Note: 
If the option “32-Bit MPLAB Harmony Project” is not visible, you need to stop and download/install MPLAB Harmony before continuing with this tutorial. 
  1. In the updated New Project dialog (Step 2. Name and Location), edit fields 1-5, depending on the hardware you are using.

  • 1) Harmony Path: Ensure that the path you enter is the path to your installation of MPLAB Harmony
  • 2) Project Location: Since this is a graphics project you can put it in the .\apps\gfx folder, but anywhere on the same drive as MPLAB Harmony will work.
  • 3) Project Name: Any project name will work, the tutorial uses “graphics_quickstart”, as long as it is all lowercase.
  • 4) Device Family: For DA and EF targets, select PIC32MZ. For the USB Starter Kit II, select PIC32MX.
  • 5) Target Board: Choose the target board associated with your hardware setup

  1. Click Finish when done. A new empty project named heartbeat will be created in MPLAB X IDE, which opens the MPLAB Harmony Configurator (MHC) plug-in.
  2. Within MHC, launch the Graphics Composer.

  1. Next, you should see the MHGC’s Welcome Dialog.
    • Click the first icon to create a new graphics design

  • If the Welcome Dialog does not appear because you’ve disabled it, in the MPLAB Harmony Graphics Composer (MHGC) screen, use the left-most icon to create a new graphics design

Note: 
You can re-enable the Welcome Dialog by using MHGC’s File > Settings > General menu:
 
  1. Since you have not enabled the Graphics Stack within MHC, you will be prompted to enable it now. Click Yes to continue.

  1. This action opens MHGC’s New Project Wizard dialog, which has four stages: Color Mode, Memory Size, Project Type, and Finish. For the Color Mode stage, select the color mode that matches your hardware.

  1. For the Memory Size stage, set the amount to be 25% of available Program Flash. Then, accept it by clicking Next.

  1. You can create a new graphics application with a “basic template” in the Project Type stage. Select Start with a new project using a basic template, as shown in the following figure, then click Next.

  1. In the last stage (i.e., Finish stage), clicking Finish will create a screen design similar to that in aria_quickstart. Try modifying the screen design by clicking on one of the three widgets shown and moving it to a new position. Or click and drag a new widget, such as a circle, from the Widget Tool Box onto the screen.

  • Optional: Use the Heap Estimator tool to reduce the application’s heap allocation
    • Launch the Heap Estimator tool from the MPLAB Harmony Graphics Configurator Tools menu
    • Click Calculate to determine the size of the heap
    • Click Set MHC Heap Value
    • Close the Heap Estimator's window
  1. From the MHC main menu, click the Generate Code icon to create this application. When prompted, click Save to save the modified graphics configuration, and then click Generate to generate the new code.

  1. Verify that you have the right compiler selected. After the project has been generated, right mouse click project and select project Properties.
  2. Be sure you have the right compiler selected. For MPLAB Harmony v2.05 it should be XC32 V1.44 or newer.

  1. Click OK to close the Project Properties dialog.
  2. Run the main project by clicking the Run Main Project icon.

  1. The display should show what was designed and the “Modify. Generate. Run” button should react to user interaction.
